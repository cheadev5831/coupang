# 쿠팡 주문목록 조회 SPA - 요구사항 정의서

**작성일**: 2026-03-01
**버전**: 1.2.0
**스택**: Quasar (Vue 3) + TypeScript + Axios

> **v1.2 변경 사유**:
> 1. 주문번호별 그룹화 제거 → 상품 전체 플랫 리스트로 표시
> 2. 조회 실행 버튼 추가 (월 선택 후 명시적 실행)

---

## 1. 프로젝트 개요

쿠팡 계정의 세션 쿠키를 입력받아 주문 내역을 월별로 조회하고,
**상품 단위**로 체크하여 전체 금액 및 체크된 상품의 금액 합산을 확인하는 SPA.

### 실사용 목적 (스프레드시트 분석 기반)
- 한 번의 주문에 여러 상품이 포함되므로 상품 단위로 선택적 집계 필요
- 업무 경비, 개인/공용 지출 구분 등 상품별 체크 후 금액 정산

---

## 2. 핵심 데이터 단위

**체크박스 단위 = 상품(Product) 1행**

| 컬럼 | 설명 |
|------|------|
| 체크박스 | 선택/해제 |
| 주문 날짜 | YYYY.MM.DD |
| 주문 번호 | 쿠팡 주문 ID (참고용, 그룹 헤더 없음) |
| 상품명 | 전체 상품명 |
| 가격 | 결제 금액 (원) |

> 같은 주문번호의 상품이 여러 개여도 별도 행으로 분리하여 나열.
> 그룹 헤더 없이 날짜 역순으로 평탄(flat)하게 표시.

---

## 3. 사용자 흐름 (User Flow)

```
[쿠키 입력 화면]
    ↓ 쿠키 저장 (localStorage)
[월 선택]
    ↓ 연도·월 선택 후 [조회] 버튼 클릭
[상품 목록 플랫 리스트 표시]
    ↓ 상품별 체크박스 선택
[체크 금액 실시간 합산 업데이트]
```

## 4. 화면 구성

### 4-1. 쿠키 설정 패널 (CookiePanel)

| 요소 | 설명 |
|------|------|
| 쿠키 입력 Input | 쿠팡 세션 쿠키 문자열 전체 입력 (textarea) |
| 저장 버튼 | localStorage 저장 |
| 초기화 버튼 | 저장된 쿠키 삭제 |
| 상태 표시 | 설정됨 ✓ / 미설정 |

**동작 규칙**
- 앱 최초 진입 시 localStorage에 쿠키가 있으면 패널 접힌 상태로 표시
- 쿠키 없으면 입력 패널 펼친 상태로 표시
- 입력 필드 마스킹 처리 (민감 정보)
- 쿠키 저장 시 자동 조회하지 않음 → 사용자가 직접 [조회] 버튼 클릭

---

### 4-2. 월 선택 + 조회 버튼 (MonthSelector)

| 요소 | 설명 |
|------|------|
| 연도 | 좌우 화살표로 이동 (기본: 현재 연도) |
| 월 탭 | 1~12월 탭 버튼 (기본: 현재 월) |
| **조회 버튼** | **선택한 연월로 API 호출 실행** |

**동작 규칙**
- 월 탭 변경만으로는 API 호출하지 않음
- [조회] 버튼 클릭 시에만 API 호출
- 조회 중에는 버튼 비활성화 + 로딩 스피너 표시
- 쿠키 미설정 상태에서 [조회] 클릭 시 쿠키 입력 패널 펼치기 + 안내 메시지 표시

---

### 4-3. 금액 집계 요약 바 (SummaryBar) — 상단 고정

| 항목 | 설명 | 예시 |
|------|------|------|
| 전체 금액 | 해당 월 전체 상품 금액 합산 | 668,210원 |
| 체크 금액 | 체크된 상품 금액 합산 | 15,460원 |
| 체크 현황 | 선택 상품 수 / 전체 상품 수 | 2 / 58 |
| 전체 선택 | 전체 상품 선택/해제 체크박스 | |

---

### 4-4. 상품 목록 (ProductList) — 플랫 리스트

그룹 헤더 없이 상품 행만 나열.

| 요소 | 설명 |
|------|------|
| 체크박스 | 상품 선택/해제 |
| 날짜 | YYYY.MM.DD |
| 주문 번호 | 쿠팡 주문 ID (참고용) |
| 상품명 | 전체 상품명 |
| 가격 | 결제 금액 (천 단위 콤마) |

**동작 규칙**
- 날짜 역순 정렬 (최신 상품 상단)
- 같은 날짜·주문번호의 상품도 별도 행으로 평탄하게 표시
- 로딩 중: 스켈레톤 UI
- 결과 없음: "해당 월에 주문 내역이 없습니다"
- 취소/반품 상품: 집계 포함, 텍스트에 취소선으로 시각적 구분

## 5. 기능 요구사항

### F-01. 쿠키 관리
- **F-01-1**: localStorage `coupang_cookie` 키에 저장
- **F-01-2**: 앱 로드 시 자동 로드 (조회는 자동 실행하지 않음)
- **F-01-3**: 초기화 시 삭제 후 목록 비우기

### F-02. 주문 조회 (명시적 실행)
- **F-02-1**: [조회] 버튼 클릭 시에만 API 호출
- **F-02-2**: Cookie 헤더에 저장된 쿠키 포함
- **F-02-3**: 응답 파싱 → 상품 행 단위 플랫 배열로 변환
- **F-02-4**: 페이지네이션 있을 경우 전체 자동 수집
- **F-02-5**: 조회 결과는 세션 내 캐싱 (같은 연월 재조회 시 API 재호출 안 함)

### F-03. 금액 집계
- **F-03-1**: 전체 금액 = 해당 월 모든 상품 가격 합산
- **F-03-2**: 체크 금액 = 체크된 상품 가격 합산
- **F-03-3**: 월 변경 시 체크 상태 초기화

### F-04. 체크박스 (상품 단위)
- **F-04-1**: 개별 상품 행 선택/해제
- **F-04-2**: 전체 선택/해제
- **F-04-3**: 같은 월 내 체크 상태 유지 (탭 전환 후 복귀 시)

## 8. 기술 제약

### CORS 처리
쿠팡 API 직접 호출 시 CORS 오류 발생.
**결정**: Quasar 개발 서버 프록시 (`/api` → `https://www.coupang.com`) 우회.
프로덕션 배포 시 별도 프록시 서버 검토 필요.
